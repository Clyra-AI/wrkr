# Buyer Action Registry Hardening Scenario

This fixture exercises the buyer-facing action registry hardening path:

- confirmed, likely, semantic-review, and context-only confidence lanes
- credential authority and standing credential posture
- purpose plus version/config metadata
- mutable endpoint semantics from MCP, OpenAPI, and route declarations
- action lineage and action surface registry output
- design-partner and customer-safe redacted report artifacts

All names, owners, repos, tokens, and providers in this fixture are fake.
