  - Regulatory exposure mapping is not emitted per tool/per regulation article at report-ready granularity.
  - Case-study generation requires anonymized export mode that preserves patterns while removing direct org/repo identifiers.
  - Benchmark section inputs (org size/industry segmentation) are not currently emitted by scan aggregation.
  - Appendix-grade tabular exports (CSV + richer JSON matrices) are not yet available as built-in command outputs.

## Exit Criteria

- Wrkr can run deterministic public sample scans at scale (>500 repos) with pagination-safe org acquisition.
- Repo/org scans can be executed in a way that materializes analyzable content for detectors (or clearly fail-closed with explicit reason).
- Wrkr can generate a single deterministic campaign summary artifact with headline metrics and methodology metadata.
- PR distribution path is complete: relevant PRs receive deterministic comment payloads in `mode=pr`.
- UAT install-path checks (including Homebrew path) are wired into release gating policy.
- Any published claim of "unreviewed production access" is blocked or downgraded when production targets are not configured.
- Report section coverage is end-to-end for the target structure (Headline, Methodology, Inventory, Privilege Map, Approval Gap, Regulatory Exposure, Case Studies, Benchmarks, Recommendations, Appendix).
- Approval ratio outputs are backed by explicit approved-list matching and reproducible classification logic.
- Methodology section fields are emitted directly from machine-readable Wrkr outputs (not manual estimation).
- Tool inventory includes deterministic category and confidence outputs suitable for external citation.
- Regulatory outputs include per-tool, per-control/article gap status suitable for compliance-table rendering.
- Anonymized and segmented exports exist for case studies and benchmark tables without leaking organization identity.
- Appendix tables are exportable in CSV/JSON with deterministic ordering and documented schemas.
